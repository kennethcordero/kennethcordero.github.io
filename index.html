    • Reflection and Learnings

    My reflection and Learnings in this course Managing Enterprise Servers is that I learned to create workflows using Infrastructure as code, evaluate the real-world enterprise and how to improve the service, and analyze tools for operating and managing enterprise servers. In prelims, our module 1 is about creating and configuring network using virtual machines. For our module 2, I discover configuring remote and local machine to connect via SSH using a KEY instead of using a password and create a public key and private key. For our Module 3, I learned to install community enterprise OS or Red Hat Linux OS and configure remote SSH connection from remote computer to CentOS/RHEL-8. For our module 4, I learned to learn on how to use the commands to make changes to remote machines and use playbook in automating ansible commands. For our module 5, I learned how to use when command in playbook for different OS distributions and apply refactoring techniques in cleaning up the playbook codes.

    In Midterm, our module 6 discussed about the targeting specific nodes and I learn how to manage the hosts individually, apply tags in selecting plays to run and manage services from remote servers using playbooks. For our module 7, I learned how to manage files in remote servers and implement roles in ansible. For our module 8,9, & 10, I learned on how to create and design a workflow that installs, configure, and manage enterprise monitoring tools using Ansible as an Infrastructure as Code tool. I learned about Nagios, Prometheus, Elastic Search, Kibana, and Logstash. In Finals, our module 11, I learned about the containerization and able to create Dockerfile and form a workflow using Ansible as Infrastructure as Code to enable Continuous Delivery process. In module 12, I learned how to Build a Sample Web App in a Docker Container. In module 13, I learned how to analyze the advantages of cloud service and evaluate different cloud deployment and service models like NTP, OpenStack packages, SQL Database, Message Queue, Memcache and Etcd. For our module 14, I learned about the OpenStack Installation Keystone, Glance, and Nova. For our module 15, I learned about the Neutron, Horizon and Cinder OpenStack. For my overall experienced for this course is that I learned and discover new things and do a lot of trial and error when there is a command or server that in the process of running program results for an error. I learned how to fix things when there is a error and able to use my logical skills to identify how to do it or how it process.
